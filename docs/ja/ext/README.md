# ARCTX 拡張機能 (Extensions) の概要

ARCTX は、最適化や問題解決のプロセスを記録するコアモデル（RunGraph）に対し、ドメイン固有の機能を追加するための各種拡張機能を提供しています。

ここでは、標準で提供されている各拡張機能の概要と使い方を説明します。

---

## 提供されている拡張機能一覧

| 拡張機能名 | ドキュメント | 主な役割 |
| :--- | :--- | :--- |
| **Git 連携 (`git`)** | [Git連携の使い方](git.md) | Git のリポジトリ、ブランチ、コミットログ、差分（Diff）情報を RunGraph に紐付ける。 |
| **ダイアグラム (`diagram`)** | [ダイアグラムの使い方](diagram.md) | Mermaid や Graphviz 形式のダイアグラム（図）を定義・描画し、ノードやステップに貼り付ける。 |
| **コマンド実行ログ (`command`)** | [コマンドログの使い方](command.md) | 外部コマンド（テスト実行など）の実行コマンド、終了コード、標準出力、標準エラー出力を記録する。 |
> 注: **添付ファイル・メディア (`asset`)** はかつて拡張機能でしたが、現在は core 標準ペイロードです（有効化不要・常時利用可）。→ [添付ファイルの使い方](asset.md)

---

## 拡張機能の有効化と管理

各 run において、拡張機能を有効化または無効化するには、以下の CLI コマンドまたは GUI 画面の `extensions` ドロップダウンメニューを使用します。

### CLI での管理

```bash
# 1. 使用可能な拡張機能の一覧と現在の有効状態を確認する
arctx ext list

# 2. 拡張機能を有効化する（例: git 拡張）
arctx ext enable git

# 3. 拡張機能を無効化する
arctx ext disable git
```

### 初期化時の同時有効化

新規に run を作成する時点で、複数の拡張機能を有効化しておくことも可能です。

```bash
arctx init req_demo --extension git --extension diagram
```
