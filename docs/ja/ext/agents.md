# エージェント連携拡張機能 (`agents` / `codex`)

`agents` 拡張機能（およびそのアダプターである `codex`）は、AI コーディングエージェント（例: Google DeepMind の Codex）の自律的な思考プロセス、ツールの呼び出し履歴、プロンプト、セッションの開始・終了イベントを自動的かつリアルタイムに ARCTX の RunGraph に記録（録画）するための拡張機能です。

---

## 主な機能

1. **ワークセッション（WorkSession）の追跡**:
   エージェントの1回の実行やセッションを、ARCTX の Lane（Lanes）モデルである `WorkSession` としてグラフに記録します。
2. **自動ステップ作成**:
   エージェントによるツールの呼び出しや、ユーザーからのプロンプト入力を、DAG のステップ（Step）として自動追加し、時系列の有向グラフに組み立てます。
3. **セッション終了の検知とノード作成**:
   セッションの正常終了（Stop/SessionEnd）を検知すると、そのセッションの最後の状態（ティップ）に `NodePayload`（セッション結果のメタデータなど）をアタッチして完了ノードをマークします。
4. **堅牢なフォールバック設計（Fail-safe）**:
   記録処理（Hook処理）にエラーが発生した場合でも、エージェント自体のメインプロセスを巻き添えにして落とさないように、デフォルトで例外を握りつぶして終了コード `0` で安全に終了します（`--strict` フラグ指定時を除く）。

---

## 仕組み

記録処理は、以下の2つのモジュールに分離されたレイヤー構造で動作しています。

1. **セッションレコーダー (`SessionRecorder`)**:
   プラットフォーム中立な記録ロジックを持つコア機能（`arctx/ext/agents/__init__.py`）。汎用的な `agent.*` ペイロードタイプや、ツールの呼び出し履歴などをグラフに書き込みます。
2. **Codex アダプター (`arctx/ext/codex/adapter.py`)**:
   特定の AI エージェント（Codexなど）のフックイベント JSON をパースし、上記の `SessionRecorder` の呼び出しに変換する変換アダプター。

---

## 使い方 (CLI)

エージェント連携は、主に CLI コマンドのフック連携として自動的に実行されます。

### 1. フック設定のインストール
エージェント（Codexなど）の環境設定に ARCTX のフックプログラムを登録します。

```bash
# Codex の設定ファイル (.codex/settings.json) にフックコマンドをマージして登録する
arctx codex install

# 特殊なPATHにある場合、コマンドパスを明示的に指定してインストールする
arctx codex install --command "/usr/local/bin/arctx"
```

### 2. フックイベントの処理 (内部動作)
フックがインストールされると、エージェントの動作時に自動的に以下のコマンドが呼び出され、標準入力（stdin）からフックイベントの JSON を受け取ってグラフに記録します。

```bash
# 標準入力からイベントJSONを読み込み、グラフに記録する（通常は自動で呼ばれます）
arctx codex hook < event.json
```
*(💡 万が一フックの処理でエラーが起きても、通常は終了ステータス `0` で正常に終了し、エージェント本体の進行を邪魔しないフェイルセーフ仕様になっています)*
*(`--strict` フラグを付けると、例外発生時に終了ステータス非ゼロでエラー終了させることができます)*
